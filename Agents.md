Core Identity:

* This is not a to-do list.
* This is not a project manager.
* This is not a Pomodoro app.
* This is not a streak tracker.
* This is not an AI chatbot.
* Habit Build is a consistency coach.

Key Product Principles:

* Optimize for consistency, not maximum productivity.
* Reduce resistance. Protect consistency. Prevent burnout.
* Never shame users for missed days.
* Focus on long-term habit building.
* Prioritize simplicity and clarity.
* Protect users from burnout.
* Focus on helping users continue after setbacks.

Psychological Architecture (from coaching conversation):

* Reframe difficulty as resistance training — discomfort is the workout.
* Normalize struggle — say "this is hard for everyone," not "you are struggling."
* Every plan has Minimum / Target / Maximum — reaching Minimum is success.
* Good days never increase tomorrow's obligation.
* Increase targets only after 4-7 days of sustained consistency, never after one day.
* Habit first, reward second (First-Then structure).
* 5-minute rule: user only commits to 5 minutes, then has permission to stop.
* For learning habits: complete once, understanding comes later.

Always follow PRODUCT.md before adding new features.
